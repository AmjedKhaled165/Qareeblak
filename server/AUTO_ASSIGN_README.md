# نظام التعيين التلقائي للمناديب 🚀

## الميزات الرئيسية

### 1. التعيين التلقائي عند إنشاء الطلب ⚡
- يتم تعيين مندوب تلقائياً لكل طلب جديد
- لا حاجة لتحديد مندوب يدوياً

### 2. توزيع الأحمال الذكي 🎯
- يتم اختيار المندوب الذي لديه **أقل عدد من الطلبات النشطة**
- الطلبات النشطة تشمل الحالات التالية:
  - `pending` (قيد الانتظار)
  - `assigned` (تم التعيين)
  - `ready_for_pickup` (جاهز للاستلام)
  - `picked_up` (تم الاستلام)
  - `in_transit` (قيد التوصيل)

### 3. الاختيار العشوائي عند التساوي 🎲
- إذا كان هناك أكثر من مندوب بنفس الحمل، يتم الاختيار عشوائياً
- مثال: إذا كان 5 مناديب لديهم 0 طلبات، سيتم اختيار واحد عشوائياً

### 4. آلية الرجوع التلقائي (Fallback) 🔄
1. **المحاولة الأولى**: البحث عن مناديب متاحين (`is_available = true`)
2. **المحاولة الثانية**: إذا لم يوجد متاحين، يتم الاختيار من **جميع** المناديب
3. **النتيجة**: لن يبقى أي طلب بدون مندوب! ✅

## كيفية العمل

### عند إنشاء طلب جديد

```javascript
// في provider-dashboard/page.tsx
const orderData = {
    customerName: "أحمد محمد",
    customerPhone: "01234567890",
    deliveryAddress: "شارع النصر",
    items: [...],
    autoAssign: true  // 👈 يفعّل التعيين التلقائي
};

const result = await apiCall('/halan/orders', {
    method: 'POST',
    body: orderData
});
```

### في السيرفر

```javascript
// server/routes/halan-orders.js
if (!effectiveCourierId) {
    // لم يتم تحديد مندوب يدوياً
    const assignedCourier = await performAutoAssign(
        newOrder.id, 
        req.user.userId, 
        req.app.get('io')
    );
    // ✅ تم التعيين تلقائياً
}
```

## أدوات الاختبار 🧪

### 1. فحص حالة النظام
```bash
node test-auto-assign.js
```
يعرض:
- قائمة المناديب وحالة توفرهم
- عدد الطلبات النشطة لكل مندوب
- الطلبات غير المعينة (إن وجدت)

### 2. إصلاح الطلبات القديمة
```bash
node fix-unassigned-orders.js
```
يقوم بـ:
- البحث عن جميع الطلبات غير المعينة
- تعيين مندوب لكل طلب تلقائياً
- تحديث سجل الطلب (order_history)

## الملاحظات المهمة 🔔

1. **لا حاجة لإعادة تشغيل السيرفر** - يستخدم `nodemon` للتحديث التلقائي
2. **الطلبات القديمة** - استخدم `fix-unassigned-orders.js` لإصلاحها
3. **الطلبات الجديدة** - يتم تعيينها تلقائياً فوراً
4. **السجل التاريخي** - يتم حفظ كل عملية تعيين في `order_history`

## استكشاف الأخطاء 🔧

### المشكلة: الطلبات ما زالت غير معينة
**الحل**: تحقق من:
```bash
# 1. هل هناك مناديب في النظام؟
SELECT COUNT(*) FROM users WHERE user_type = 'partner_courier';

# 2. هل السيرفر يعمل؟
Get-Process | Where-Object {$_.ProcessName -like "*node*"}

# 3. تشغيل الإصلاح اليدوي
node fix-unassigned-orders.js
```

### المشكلة: خطأ "لم يتم العثور على أي مناديب"
**الحل**: أضف مناديب للنظام عبر لوحة التحكم
