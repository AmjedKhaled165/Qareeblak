# 🐛 دليل تصحيح الأخطاء - Debugging Guide

## لإصلاح مشكلة "حدث خطأ في جلب بيانات الحجز"

### الخطوات:

1. **افتح أدوات المطور**
   - اضغط على `F12` من لوحة المفاتيح
   - أو اضغط بزر الفأرة الأيمن → اختر "Inspect" أو "فحص العنصر"

2. **انتقل إلى لسان Console (الكونسول)**
   - هناك أيقونة "Console" في الأعلى

3. **احذر الـ Console من الرسائل القديمة**
   - اضغط على أيقونة المسح (🗑️) أو اكتب `clear()` واضغط Enter

4. **قم بتنفيذ الخطوات التي تسبب الخطأ:**
   - اذهب إلى صفحة الخدمات
   - اختر خدمة وأنشئ طلب
   - اضغط على "تتبع الطلب"

5. **لاحظ الرسائل في Console:**
   - ستظهر رسائل زرقاء تبدأ بـ "🔵 Fetching order with ID:"
   - ستظهر رسائل حمراء تبدأ بـ "❌ Failed to fetch order"
   - ستظهر رسائل خطأ حمراء تبدأ بـ "📛 Error Message:"

6. **انسخ جميع الرسائل والأخطاء:**
   - حدد كل الرسائل (Ctrl+A)
   - انسخها (Ctrl+C)
   - الصقها في رسالة

## ماذا تبحث عنه:

### الرسالة الناجحة:
```
🔵 Fetching order with ID: abc123def456
========== ✅ FULL ORDER DATA ==========
Complete Order Object: {id: "abc123def456", source: "qareeblak", ...}
Order Keys: ["id", "source", "channel", "serviceName", ...]
Source: "qareeblak"
```

### الرسالة الخاطئة:
```
❌ Failed to fetch order - Full Error:
Error: 404 Not Found (أو error message آخر)
📛 Error Message: حدث خطأ في جلب بيانات الحجز
ID: abc123def456
```

---

## المشاكل المحتملة وحلولها:

### 1️⃣ الخطأ: "404 Not Found"
**الحل**: معرّف الطلب غير صحيح أو لم يتم حفظه بشكل صحيح
- تأكد من أن الطلب تم إنشاؤه بنجاح
- تحقق من معرّف الطلب في قاعدة البيانات

### 2️⃣ الخطأ: "401 Unauthorized"  
**الحل**: التوكن (token) لم يتم حفظه أو انتهى صلاحيته
- جرب تسجيل الدخول مرة أخرى
- امسح الـ localStorage وسجل الدخول من جديد

### 3️⃣ الخطأ: "500 Internal Server Error"
**الحل**: مشكلة في السيرفر (الخادم)
- تحقق من أن الخادم يعمل بشكل صحيح
- تحقق من سجلات الخادم

### 4️⃣ Timeout أو لا توجد رسالة خطأ
**الحل**: الخادم لا يستجيب
- تأكد من أن الخادم قيد التشغيل على `http://localhost:5000`

### 5️⃣ معرّف الطلب فارغ أو غريب
**الحل**: مشكلة في إنشاء الطلب
- تحقق من أن الطلب تم إنشاؤه بنجاح
- تحقق من رسائل الخطأ عند الإنشاء

## استخدام Script التصحيح المتقدم:

### الطريقة الأولى: النسخ واللصق
1. افتح أدوات المطور (F12)
2. اذهب إلى Console
3. افتح ملف `DEBUG_SCRIPT.js` من المشروع
4. انسخ محتوى الملف كاملاً
5. الصقه في Console واضغط Enter
6. ستحصل على تقرير كامل عن حالة الـ API

### البيانات التي يطبعها Script:
- ✅ معرّف الطلب من URL
- ✅ حالة التوكنات (موجودة أم لا)
- ✅ URL الـ API
- ✅ طلب API الفعلي والرد
- ✅ بيانات الطلب الكاملة (إن كانت موجودة)

---

## المعلومات المطلوبة للإبلاغ عن الخطأ:

عند الإبلاغ عن المشكلة، أرسل:

1. **لقطة من Console** تتضمن:
   - الرسائل الزرقاء (🔵 Fetching)
   - الرسائل الحمراء (❌ Failed)
   - الرسائل الخطأ (📛 Error Message)

2. **معرّف الطلب (Order ID)**

3. **URL الطلب الكامل** (من شريط العنوان)

4. **رسالة الخطأ كاملة** (مثل 404, 401, 500)

5. **هل تم تسجيل الدخول؟** (بـ Customer أم Partner؟)

---

## نصائح مهمة:

- 🔄 حاول تحديث الصفحة (F5) وكرر الخطوات
- 🔄 امسح Cache من أدوات المطور (Clear All)
- 📱 جرب في متصفح آخر (Chrome, Firefox, Edge)
- 🔐 تأكد من تسجيل الدخول الصحيح
- 🔐 تأكد من عدم انتهاء صلاحية التوكن
